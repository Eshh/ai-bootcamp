1. Variables & Data Types

Basic Variable Assignment

Python is dynamically typed ‚Äî you don't declare a type; it's inferred from the value.

# Basic assignments
text = "Hello"        # str
number = 42           # int
pi = 3.14             # float
is_active = True      # bool
üí°
Pro Tip: Use type() to inspect any variable's type at runtime: type(pi) ‚Üí <class 'float'>

Type Coercion vs. Type Conversion

Coercion (implicit) ‚Äî Python auto-converts in mixed expressions:

5 + 2.0       # int + float ‚Üí float: 7.0
True + 1      # bool + int ‚Üí int: 2

Conversion (explicit) ‚Äî You manually cast:

int("42")      # str ‚Üí int: 42
float("3.14")  # str ‚Üí float: 3.14
str(100)       # int ‚Üí str: "100"
bool(0)        # int ‚Üí bool: False
bool("")       # str ‚Üí bool: False (empty = falsy)
‚ö†Ô∏è
Pitfall: int("3.14") raises ValueError. Convert via: int(float("3.14"))
üí°
Going Deeper ‚Äî Truthy & Falsy: These are False in boolean context: 0, 0.0, "", [], {}, (), set(), None. Everything else is True. Idiomatic: if my_list: instead of if len(my_list) > 0:

3. String Formatting

Python offers three ways to format strings. f-strings are the modern standard.

f-Strings (Python 3.6+ ‚Äî Recommended) ‚≠ê

name = "Eswar"
age = 30
print(f"My name is {name} and I am {age} years old.")

# Expressions inside braces
print(f"Next year I'll be {age + 1}")
print(f"Name uppercase: {name.upper()}")

# Number formatting
pi = 3.14159
print(f"Pi: {pi:.2f}")           # 3.14
print(f"Big: {1000000:,}")        # 1,000,000
print(f"Pct: {0.856:.1%}")       # 85.6%

.format() Method

print("Hello, {}! Age: {}".format(name, age))
print("Hello, {0}! {0} is {1}".format(name, age))   # Positional
print("Hello, {n}!".format(n=name))                 # Named

% Formatting (Legacy ‚Äî Avoid)

print("Hello, %s! Age: %d" % (name, age))
print("Pi: %.2f" % pi)
SpecifierTypeExample
%sString"Hi %s" % "world"
%dInteger"Age: %d" % 25
%fFloat"Pi: %.2f" % 3.14
%xHex"%x" % 255 ‚Üí ff
üí°
Best Practice: Always use f-strings for new code ‚Äî fastest, most readable, most Pythonic. Use .format() only when the template string is stored externally (e.g., config file).

Lists ‚Äî Ordered, Mutable Sequences

fruits = ["apple", "banana", "cherry"]

# Access & Slice
fruits[0]             # "apple"
fruits[-1]            # "cherry" (last)
fruits[1:3]           # ["banana", "cherry"]

# Modify
fruits.append("date")
fruits.insert(1, "avocado")
fruits.remove("banana")
popped = fruits.pop()

# List comprehensions (Pythonic!)
squares = [x**2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]

Dictionaries ‚Äî Key-Value Mappings

person = {
    "name": "Eswar",
    "age": 30,
    "skills": ["Python", "AI"]
}

person["name"]                # "Eswar"
person.get("email", "N/A")    # "N/A" (safe access)

for key, value in person.items():
    print(f"{key}: {value}")
‚ö†Ô∏è
Pitfall: person["missing"] raises KeyError. Always use .get() for safe access.

Tuples ‚Äî Immutable Sequences

coords = (10.5, 20.3)
x, y = coords        # Tuple unpacking

# Single-element tuple (note the comma!)
single = (42,)       # Tuple
not_tuple = (42)     # Just an int!
üí°
When to use: Tuples for fixed data (coordinates, RGB). They're faster than lists and can be dict keys.

Sets ‚Äî Unordered, Unique Elements

nums = {1, 2, 3, 3, 2}  # {1, 2, 3}

a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b    # Union: {1, 2, 3, 4, 5, 6}
a & b    # Intersection: {3, 4}
a - b    # Difference: {1, 2}
a ^ b    # Symmetric Diff: {1, 2, 5, 6}
üí°
Best Practice: x in my_set is O(1) vs O(n) for lists. Use sets for fast membership testing.
‚ö†Ô∏è
Pitfall: {} creates an empty dictionary, not a set. Use set() for an empty set.

6. Magic Comments

Special comments that affect how Python interprets the source file.

Encoding Declaration

# -*- coding: utf-8 -*-

Declares the file's character encoding. Rarely needed in Python 3 (UTF-8 is default).

Shebang Line

#!/usr/bin/env python3

Tells Unix/macOS which interpreter to use when running the script directly.

Other Common Special Comments

CommentPurpose
# type: ignoreSkip type checking (mypy/pyright)
# noqaSkip linting (flake8)
# pragma: no coverExclude from test coverage
# TODO:Mark planned work (IDE-recognized)
# FIXME:Mark known bugs (IDE-recognized)

‚úÖ Key Takeaways

  1. Python is dynamically typed ‚Äî types are inferred, not declared
  2. Multiple assignment and tuple unpacking are Pythonic patterns you should use
  3. Use f-strings for all string formatting in modern Python
  4. math.pi is a constant (no parentheses); use ** over math.pow() for integer powers
  5. Lists = mutable sequences, Tuples = immutable, Dicts = key-value, Sets = unique & fast lookup
  6. Understand type coercion (implicit) vs type conversion (explicit)
  7. Know your falsy values: 0, 0.0, "", [], {}, None, False
